## 1. 接口描述

创建数据安全策略，私有化调用path为：capi/DLPOpenApi/CreateDLPPolicy 8.6P1版本开始支持

## 2. 输入参数

| 参数名称 | 必选  | 类型 | 描述                                  |
|---------|-----|---------|-------------------------------------|
| Name | 是   | String | 策略名称;Required:true<br/>示例值：扫描       |
| Description | 否   | String | 描述<br/>示例值：扫描办公设备                   |
| Status | 否   | Integer | 启用停用状态 0停用 1启用(只支持32位)<br/>示例值：1    |
| PolicyType | 是   | Integer | 策略类型, DLP 策略类型值固定为 7|
| PolicySubType | 否   | Integer | 策略子类型,告警策略:43;管控范围策略:44;截屏取证策略:45;拦截策略:46;自定义通道:47;通道白名单:48|

## 3. 输出参数

| 参数名称 | 类型 | 描述 |
|---------|---------|---------|
| Data | [CreateDLPPolicyRspData](/开放API/云规范接口/版本：2022-06-01/数据结构.md#CreateDLPPolicyRspData) | 业务响应数据<br/>注意：此字段可能返回 null，表示取不到有效值。|
| RequestId | String | 唯一请求 ID，由服务端生成，每次请求都会返回（若请求因其他原因未能抵达服务端，则该次请求不会获得 RequestId）。定位问题时需要提供该次请求的 RequestId。|

## 4. 示例

### 示例1 创建数据安全策略

#### 输入示例

```
POST / HTTP/1.1
Host: ioa.tencentcloudapi.com
Content-Type: application/json
X-TC-Action: CreateBackendPolicy
<公共请求参数>

{
  "Name": "test",
  "Status": 1,
  "Description": "测试",
  "PolicyType":7,
  "PolicySubType":43
}
```

#### 输出示例

```json
{
    "Response": {
        "Data": {
            "PolicyId": 49
        },
        "RequestId": "web-1245921911577-1765938455417"
    }
}
```
